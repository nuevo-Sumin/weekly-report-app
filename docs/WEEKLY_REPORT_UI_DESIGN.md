# 주간보고 화면 구성 설계

## 1. 설계 기준

이 문서는 주간업무보고 앱의 화면 구성을 정의한다.

역할 표현은 다음처럼 매핑한다.

- 개발자(팀원): 일반사용자, `MEMBER`
- PL(팀장): 관리자, `ADMIN`

핵심 흐름:

1. 사용자는 메인페이지에서 로그인한다.
2. 최초 사용자는 회원가입 시 직급을 선택한다.
3. 개발자(팀원)는 주간업무를 입력하거나 CSV 파일로 업로드한다.
4. 개발자(팀원)는 제출할 행을 체크박스로 선택하고 제출/저장한다.
5. PL(팀장)은 팀원별 제출 내용을 조회하고 필요한 항목을 선택한다.
6. PL(팀장)은 단위업무별로 병합한 결과를 수정/복사한다.

## 2. 전체 화면 목록

| 화면 ID | 화면명 | 접근 권한 | 주요 목적 |
| --- | --- | --- | --- |
| A | 메인페이지 | 비로그인 | 로그인, 회원가입, 계정 찾기 진입 |
| A-1 | 회원가입페이지 | 비로그인 | 계정 생성 및 직급 선택 |
| A-2 | 아이디/패스워드 찾기 | 비로그인 | 이메일 기반 아이디 안내 및 패스워드 재설정 |
| B | 팀원 제출 화면 | 개발자(팀원) | 주간업무 작성, 업로드, 미리보기, 저장/제출 |
| C | 팀장 취합 화면 | PL(팀장) | 팀원 제출 내용 조회, 병합, 최종 텍스트 복사 |

## 3. 공통 레이아웃

로그인 후 화면은 업무 도구형 UI로 구성한다.

상단 영역:

- 서비스명: 주간업무보고
- 로그인 사용자명
- 직급 표시
- 로그아웃 버튼

본문 영역:

- 화면 제목
- 보고 기준 주차/기간 표시
- 주요 액션 버튼
- 그리드 또는 입력 영역
- 미리보기/병합 결과 패널

상태 메시지:

- 저장 성공
- 임시저장 성공
- 제출 완료
- 업로드 성공/실패
- 복사 완료
- 필수 입력 누락

## 4. A. 메인페이지

### 4.1 목적

사용자가 로그인하거나 회원가입/계정 찾기 화면으로 이동하는 첫 화면이다.

### 4.2 화면 구성

중앙 로그인 패널:

- 아이디 입력
- 패스워드 입력
- 아이디 기억하기 체크박스
- 로그인 버튼
- 회원가입 버튼
- 아이디/패스워드 찾기 링크 또는 버튼

### 4.3 컨트롤 상세

| 컨트롤 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| 아이디 | text input | Y | 로그인 ID 입력 |
| 패스워드 | password input | Y | 비밀번호 입력 |
| 아이디 기억하기 | checkbox | N | 체크 시 다음 방문 때 아이디 자동 표시 |
| 로그인 | button | Y | 인증 요청 |
| 회원가입 | button | N | 회원가입페이지 이동 |
| 아이디/패스워드 찾기 | link/button | N | 계정 찾기 화면 이동 |

### 4.4 동작

- 아이디 또는 패스워드가 비어 있으면 로그인 요청을 보내지 않고 안내 메시지를 표시한다.
- 로그인 성공 시 role에 따라 이동한다.
  - 개발자(팀원): 팀원 제출 화면
  - PL(팀장): 팀장 취합 화면
- 아이디 기억하기 체크 시 브라우저에 아이디만 저장한다.
- 패스워드는 브라우저에 저장하지 않는다.

### 4.5 예외

- 아이디가 존재하지 않으면 오류 메시지를 표시한다.
- 패스워드가 틀리면 오류 메시지를 표시한다.
- 비활성 계정이면 관리자 문의 메시지를 표시한다.

## 5. A-1. 회원가입페이지

### 5.1 목적

신규 사용자가 계정을 생성하고 직급을 선택한다.

### 5.2 화면 구성

회원가입 폼:

- 아이디
- 패스워드
- 패스워드 확인
- 이메일
- 직급 선택
  - 개발자(팀원)
  - PL(팀장)
- 회원가입 버튼
- 취소 또는 로그인으로 돌아가기 버튼

### 5.3 컨트롤 상세

| 컨트롤 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| 아이디 | text input | Y | 로그인에 사용할 고유 ID |
| 패스워드 | password input | Y | 비밀번호 |
| 패스워드 확인 | password input | Y | 비밀번호 재입력 |
| 이메일 | email input | Y | 아이디 안내/패스워드 재설정에 사용 |
| 직급 | radio/select | Y | 개발자(팀원), PL(팀장) 중 선택 |
| 회원가입 | button | Y | 계정 생성 |
| 취소 | button | N | 메인페이지로 이동 |

### 5.4 동작

- 아이디 중복 여부를 확인한다.
- 패스워드와 패스워드 확인이 다르면 저장하지 않는다.
- 이메일 형식이 올바르지 않으면 저장하지 않는다.
- 직급을 선택하지 않으면 저장하지 않는다.
- 회원가입 성공 후 메인페이지로 이동하거나 자동 로그인할 수 있다.

### 5.5 권한 메모

PL(팀장) role을 회원가입 시 바로 허용하면 누구나 관리자 권한을 가질 수 있다. 운영 단계에서는 PL 가입 후 승인 대기 상태를 두는 방식을 권장한다.

## 6. A-2. 아이디/패스워드 찾기

### 6.1 목적

사용자가 이메일을 통해 아이디를 안내받거나 패스워드를 재설정한다.

### 6.2 화면 구성

탭 또는 섹션:

- 아이디 찾기
- 패스워드 재설정

아이디 찾기:

- 이메일 입력
- 아이디 안내 요청 버튼

패스워드 재설정:

- 아이디 입력
- 이메일 입력
- 재설정 메일 발송 버튼

### 6.3 동작

- 아이디 찾기는 가입된 이메일로 아이디 안내 메일을 발송한다.
- 패스워드 찾기는 이메일로 패스워드 재설정 링크를 발송한다.
- 보안상 화면에는 전체 아이디나 가입 여부를 과도하게 노출하지 않는다.
- 패스워드는 이메일로 직접 보내지 않고 재설정 링크만 보낸다.

## 7. B. 팀원 제출 화면

### 7.1 목적

개발자(팀원)가 주간업무 내용을 직접 입력하거나 CSV로 업로드하고, 제출할 항목을 선택해 저장/제출하는 화면이다.

### 7.2 화면 레이아웃

상단 조건 영역:

- 기준 일자 선택
- 자동 계산된 주차 표시
- 보고 기간 표시
- 전주 자료 불러오기 버튼
- CSV 업로드 버튼
- metabuild PMS 버전으로 생성하기 버튼

중앙 작업 영역:

- 업무 항목 그리드
- 행 추가/삭제 버튼
- 행별 체크박스

우측 또는 하단 미리보기 영역:

- 미리보기 창
- 금주/차주 구분 출력
- 단위업무별 병합 결과

하단 액션 영역:

- 임시저장
- 제출/저장

### 7.3 상단 조건 영역

| 컨트롤 | 타입 | 설명 |
| --- | --- | --- |
| 일자 선택 | date picker | 선택한 날짜 기준으로 주차와 보고 기간 자동 계산 |
| 주차 표시 | readonly text | 예: 2026년 22주차 |
| 보고 기간 표시 | readonly text | 예: 2026.05.25 ~ 2026.05.29 |
| CSV 업로드 | file button | `.csv` 업로드 |
| 전주 자료 불러오기 | button | 이전 주 업무 항목을 현재 주차 초안으로 복사 |
| metabuild PMS 버전으로 생성하기 | button | metabuild PMS 형식의 텍스트 미리보기 생성 |

### 7.4 주차 자동 계산

- 사용자가 일자를 선택하면 해당 날짜가 포함된 주의 월요일~금요일을 계산한다.
- 계산 결과를 보고 기간으로 표시한다.
- 내부 저장 시 `report_start_date`, `report_end_date`로 사용한다.
- 일자를 변경하면 현재 작성 중인 항목의 보고 기간도 변경할지 확인한다.

### 7.5 CSV 업로드

지원 방식:

- CSV 양식에는 주차 구분 컬럼을 두지 않는다.
- 업로드 후 화면에서 각 row별로 `금주`, `차주`, `모두` 중 하나를 선택한다.

업로드 후 동작:

- CSV row를 업무 항목 그리드로 변환한다.
- 필수 컬럼 누락 시 오류를 표시한다.
- 변환 실패 row는 저장 결과 영역에 검증 오류로 표시하고, 정상 row는 계속 검토/저장할 수 있게 한다.
- 업로드한 항목은 바로 제출하지 않고 그리드에서 검토 후 저장/제출한다.

### 7.6 전주 자료 불러오기

동작:

- 로그인 사용자 기준 전주 보고 기간의 저장/제출 항목을 조회한다.
- 조회된 항목을 현재 주차의 초안으로 복사한다.
- 완료 상태, 진행률, 진행내용은 그대로 둘지 초기화할지 정책 결정이 필요하다.

권장 복사 정책:

- 단위업무, 세부사항은 유지
- 진행내용은 유지하되 사용자가 수정 가능
- 완료기한은 현재 주차 기준으로 재설정 가능
- 제출 상태는 임시저장 상태로 초기화

### 7.7 업무 항목 그리드

그리드는 행별 선택이 가능해야 한다.

권장 컬럼:

| 컬럼 | 입력 방식 | 필수 | 설명 |
| --- | --- | --- | --- |
| 선택 | checkbox | N | 제출/병합 대상 선택 |
| 주차 구분 | select | Y | 금주, 차주 |
| 단위업무 | select/text | Y | 단위업무 분류 |
| 세부사항 | textarea/text | Y | 업무 상세 |
| 진행내용 | textarea/text | Y | 금주 진행 또는 차주 예정 |
| 완료기한 | date picker | N | 완료 예정일 |
| 완료여부 | checkbox/select | N | 완료, 미완료 |
| 진행률 | number/slider | N | 0~100 |
| 상태 | select | N | 신규, 진행중, 완료, 보류 |
| 비고 | text | N | 추가 메모 |

그리드 기능:

- 행 추가
- 행 삭제
- 행 복사
- 다중 선택
- 선택 항목만 미리보기 반영
- 필수값 누락 행 강조
- 금주/차주 필터
- 단위업무 필터

### 7.8 미리보기 창

미리보기 창은 선택된 항목을 기준으로 생성한다.

표시 구조:

```text
## 주간업무보고 (YYYY.MM.DD ~ YYYY.MM.DD)

### 금주

[단위업무]
- 세부사항: 진행내용

### 차주

[단위업무]
- 세부사항: 진행내용
```

동작:

- 행 선택, 입력값 변경, 주차 구분 변경 시 미리보기를 갱신한다.
- 단위업무별로 항목을 그룹화한다.
- 금주와 차주는 서로 섞이지 않게 분리한다.
- metabuild PMS 버전 생성 시 별도 템플릿으로 미리보기를 갱신한다.

### 7.9 임시저장

동작:

- 현재 그리드의 모든 행을 임시저장한다.
- 체크 여부도 함께 저장한다.
- 필수값이 일부 비어 있어도 임시저장은 허용할 수 있다.
- 임시저장 후 상태 메시지를 표시한다.

### 7.10 제출/저장

동작:

- 체크된 행만 제출 대상으로 저장한다.
- 필수값 누락 행이 있으면 제출을 막는다.
- 제출 후 해당 항목은 팀장 취합 화면에서 조회 가능해야 한다.
- 제출 후 수정 가능 여부는 정책으로 결정한다.

권장 정책:

- 저장: 본인 화면에서 계속 수정 가능
- 제출: 팀장 화면에 노출
- 제출 후 수정: 수정 가능하되 수정일시와 제출 상태 변경 이력을 남김

### 7.11 metabuild PMS 버전으로 생성하기

목적:

- 현재 선택된 업무 항목을 metabuild PMS에 붙여 넣기 쉬운 텍스트 형식으로 변환한다.

동작:

- 선택된 행만 대상으로 한다.
- 금주/차주와 단위업무를 반영한다.
- 기존 프로토타입의 보고서 생성 로직을 복구/개선해 사용할 수 있다.
- 생성 결과는 미리보기 창에 표시한다.

## 8. C. 팀장 취합 화면

### 8.1 목적

PL(팀장)이 팀원들이 제출한 보고 내용을 조회하고, 필요한 행을 선택해 단위업무별로 병합한 뒤 최종 텍스트를 복사하는 화면이다.

### 8.2 화면 레이아웃

상단 조건 영역:

- 기준 일자 선택
- 자동 계산된 주차 표시
- 보고 기간 표시
- 팀원 필터
- 단위업무 필터
- 제출 상태 필터
- 전주 자료 불러오기 버튼

중앙 작업 영역:

- 팀원별 보고 내용 그리드
- 행별 체크박스
- 단위업무별 병합 버튼

우측 또는 하단 결과 영역:

- 병합 결과 미리보기
- 결과 직접 수정 textarea
- 최종 텍스트 복사 버튼

### 8.3 팀원별 보고 내용 조회

조회 조건:

- 보고 기간
- 팀원
- 단위업무
- 금주/차주
- 제출 상태

기본 조회:

- 현재 주차의 제출 완료 항목
- 모든 팀원
- 모든 단위업무

### 8.4 팀장 그리드

권장 컬럼:

| 컬럼 | 입력 방식 | 설명 |
| --- | --- | --- |
| 선택 | checkbox | 병합 대상 선택 |
| 팀원 | readonly text | 제출자 |
| 주차 구분 | readonly/select | 금주, 차주 |
| 단위업무 | readonly/select | 단위업무 |
| 세부사항 | readonly/editable | 업무 상세 |
| 진행내용 | readonly/editable | 진행 또는 예정 내용 |
| 완료기한 | readonly/editable | 완료 예정일 |
| 완료여부 | readonly/editable | 완료 상태 |
| 제출일시 | readonly text | 팀원 제출 시간 |
| 수정일시 | readonly text | 마지막 수정 시간 |

편집 정책:

- 기본은 조회 전용으로 둔다.
- 팀장이 병합 전 문구를 다듬어야 한다면 특정 컬럼만 인라인 편집 가능하게 한다.
- 원본 팀원 제출 내용과 팀장 수정 내용은 구분 저장하는 것이 좋다.

### 8.5 전주 자료 불러오기

동작:

- 전주 팀장 병합 결과 또는 전주 제출 항목을 불러온다.
- 현재 주차 취합 초안으로 사용할 수 있게 한다.
- 불러온 항목은 바로 최종본이 아니라 편집 가능한 초안 상태로 둔다.

사용 목적:

- 반복 업무가 많은 경우 취합 양식을 빠르게 시작
- 전주 미완료 항목을 차주/금주 항목으로 이어서 관리

### 8.6 항목 체크박스 선택

동작:

- 체크된 항목만 병합 대상이 된다.
- 전체 선택/해제 기능을 제공한다.
- 팀원별 선택/해제 기능을 제공하면 취합 작업이 빨라진다.
- 단위업무별 선택/해제 기능도 제공할 수 있다.

### 8.7 단위업무별 병합

병합 기준:

1. 금주/차주
2. 단위업무
3. 팀원 또는 정렬 순서

병합 규칙:

- 선택된 항목만 포함한다.
- 금주와 차주는 별도 섹션으로 분리한다.
- 같은 단위업무 항목은 하나의 묶음으로 표시한다.
- 팀원 이름을 표시할지 여부는 옵션으로 둘 수 있다.

권장 병합 출력:

```text
## 팀 주간업무보고 (YYYY.MM.DD ~ YYYY.MM.DD)

### 금주

[단위업무]
- 홍길동: 세부사항 / 진행내용
- 김개발: 세부사항 / 진행내용

### 차주

[단위업무]
- 홍길동: 세부사항 / 예정내용
```

### 8.8 병합 결과 미리보기

구성:

- 자동 병합 결과 표시
- 직접 수정 가능한 textarea
- 원본 재생성 버튼
- 최종 텍스트 복사 버튼

동작:

- 병합 버튼 클릭 시 미리보기 내용을 갱신한다.
- 사용자가 미리보기 텍스트를 직접 수정할 수 있다.
- 직접 수정 후 복사하면 수정된 텍스트가 복사된다.
- 최종본 저장 기능이 추가될 경우 수정된 텍스트를 `MergedReport`에 저장한다.

### 8.9 최종 텍스트 복사 버튼

동작:

- 병합 결과 미리보기의 현재 텍스트를 클립보드에 복사한다.
- 복사 성공 메시지를 표시한다.
- 필요하면 마지막 복사일시를 저장한다.

## 9. 화면 간 이동

비로그인:

- 메인페이지
- 회원가입페이지
- 아이디/패스워드 찾기

로그인 후:

- 개발자(팀원): 팀원 제출 화면으로 이동
- PL(팀장): 팀장 취합 화면으로 이동

권장 내비게이션:

- 팀원 제출
- 팀장 취합
- 내 정보
- 로그아웃

PL(팀장)은 팀원 제출 화면도 접근 가능하게 할 수 있다. 단, 본인의 팀원 역할 제출 기능과 관리자 취합 기능을 명확히 구분해야 한다.

## 10. 상태 및 검증 규칙

필수 검증:

- 로그인: 아이디, 패스워드 필수
- 회원가입: 아이디, 패스워드, 이메일, 직급 필수
- 팀원 제출: 주차 구분, 단위업무, 세부사항, 진행내용 필수
- CSV 업로드: 필수 컬럼 검증
- 팀장 병합: 최소 1개 이상 항목 선택 필요

상태값:

- 임시저장
- 저장
- 제출
- 병합 초안
- 최종 병합

## 11. 구현 우선순위

1차 구현:

- 메인페이지
- 회원가입페이지
- 팀원 제출 화면
- 일자 선택 및 주차 자동 계산
- 수동 입력 그리드
- 임시저장/제출
- 미리보기

2차 구현:

- CSV 업로드
- 전주 자료 불러오기
- metabuild PMS 버전 생성
- 팀장 취합 화면
- 단위업무별 병합
- 최종 텍스트 복사

3차 구현:

- 아이디/패스워드 찾기
- 이메일 발송
- 팀장 편집 이력
- 업로드 실패 row 상세 관리
- 병합 결과 버전 관리

## 12. 구현 전 결정 필요 사항

- PL(팀장) 가입을 즉시 승인할지 승인 대기 상태를 둘지
- CSV 업로드 후 금주/차주/모두 선택 UX를 어느 수준까지 보강할지
- 팀원 제출 후 수정 가능 범위
- 팀장 화면에서 원본 항목을 직접 수정할지, 병합 결과 텍스트만 수정할지
- 완료기한과 완료여부를 동시에 둘지, 업무 상태로 통합할지
- metabuild PMS 버전 템플릿의 정확한 출력 형식
