# 주간업무보고 앱 요구사항

## 1. 목적

팀원이 주간업무 내용을 직접 입력하거나 CSV/Excel 파일로 업로드한 뒤, 제출할 항목을 선택하고 단위업무별로 병합된 주간업무보고 텍스트를 생성하는 웹 애플리케이션을 만든다.

관리자는 팀원이 제출한 업무 항목을 조회하고, 단위업무 기준으로 병합/수정한 최종 보고 텍스트를 생성 및 복사할 수 있어야 한다.

## 2. 사용자 역할

### 일반사용자

일반사용자는 팀원을 의미한다.

- 최초 회원가입 시 `일반사용자` role을 선택할 수 있다.
- 로그인 후 본인의 주간업무 내용을 작성, 임시저장, 저장, 수정, 조회할 수 있다.
- 업무 항목을 직접 입력할 수 있다.
- CSV/Excel 파일을 업로드해 업무 항목을 일괄 등록할 수 있다.
- 제출할 업무 항목을 체크박스로 선택할 수 있다.
- 선택한 항목을 단위업무별로 병합할 수 있다.
- 병합된 결과를 텍스트 형태로 확인할 수 있다.
- 최종 텍스트를 클립보드에 복사할 수 있다.

### 관리자

관리자는 팀원들이 제출한 주간업무 항목을 취합하는 사용자를 의미한다.

- 최초 회원가입 시 `관리자` role을 선택할 수 있다.
- 로그인 후 팀원이 제출한 업무 항목을 조회할 수 있다.
- 팀원별, 기간별, 단위업무별 업무 항목을 불러올 수 있다.
- 제출된 업무 항목을 단위업무별로 병합할 수 있다.
- 병합 결과를 직접 수정할 수 있다.
- 수정된 최종 보고 텍스트를 클립보드에 복사할 수 있다.

## 3. 인증 및 권한 요구사항

- 사용자는 회원가입을 할 수 있어야 한다.
- 회원가입 시 role을 선택해야 한다.
  - 일반사용자
  - 관리자
- 사용자는 로그인 후에만 주간업무 작성 및 조회 기능을 사용할 수 있다.
- 로그인하지 않은 사용자는 작성, 저장, 조회, 병합 기능에 접근할 수 없다.
- 일반사용자는 본인이 작성한 업무 항목만 조회/수정할 수 있다.
- 관리자는 모든 팀원의 제출 항목을 조회할 수 있다.
- 관리자 전용 화면 또는 기능은 관리자 role 사용자만 접근할 수 있다.

## 4. 주간업무 입력 요구사항

일반사용자는 주간업무 항목을 직접 입력할 수 있어야 한다.

업무 항목의 기본 정보는 다음과 같다.

- 단위업무
- 업무 제목
- 업무 상세 내용
- 진행 내용
- 진행 상태
- 진행률
- 작성자
- 대상 주차 또는 보고 기간
- 주차 구분
  - 금주
  - 차주
- 작성일시
- 수정일시
- 제출 여부

진행 상태 예시는 다음과 같다.

- 신규
- 진행중
- 완료
- 보류

## 5. CSV/Excel 업로드 요구사항

일반사용자는 CSV 또는 Excel 파일을 업로드해 업무 항목을 등록할 수 있어야 한다.

- CSV/Excel 업로드는 일반사용자 화면에서 제공한다.
- 업로드된 CSV/Excel의 각 row는 업무 항목으로 변환된다.
- CSV/Excel 항목은 단위업무 기준으로 분류되어야 한다.
- Excel 파일의 데이터는 금주 업무와 차주 업무로 구분해 입력할 수 있어야 한다.
- 금주/차주 구분은 다음 방식 중 하나를 지원할 수 있다.
  - `주차 구분` 컬럼에 `금주` 또는 `차주` 값을 입력
  - Excel 시트를 `금주`, `차주`로 분리
  - 업로드 후 화면에서 각 항목의 금주/차주 구분을 수정
- 업로드 후 사용자는 변환된 항목 목록을 확인할 수 있어야 한다.
- 사용자는 변환된 항목 중 제출할 항목을 체크박스로 선택할 수 있어야 한다.
- CSV/Excel 헤더가 올바르지 않거나 필수 컬럼이 누락되면 오류 메시지를 표시해야 한다.
- 업로드된 항목은 바로 최종 제출되지 않고, 사용자가 확인 후 저장 또는 제출할 수 있어야 한다.

권장 CSV/Excel 필수 컬럼:

- 주차 구분
- 단위업무
- 업무 제목
- 업무 상세 내용
- 진행 내용
- 진행 상태
- 진행률

## 6. 항목 선택 및 병합 요구사항

일반사용자는 저장된 업무 항목 목록에서 제출할 항목을 체크박스로 선택할 수 있어야 한다.

- 체크된 항목만 병합 대상이 된다.
- 선택된 항목은 단위업무별로 그룹화된다.
- 같은 단위업무에 속한 항목은 하나의 묶음으로 병합된다.
- 병합 시 금주 업무와 차주 업무는 서로 섞이지 않고 구분되어 출력되어야 한다.
- 병합 결과는 텍스트 형태로 출력된다.
- 병합 텍스트는 사용자가 복사하기 전에 확인할 수 있어야 한다.
- 일반사용자는 병합 결과를 복사할 수 있어야 한다.

예상 출력 형식:

```text
## 주간업무보고 (YYYY.MM.DD ~ YYYY.MM.DD)

### 금주

[단위업무명]
- 업무 제목 1: 진행 내용
- 업무 제목 2: 진행 내용

### 차주

[단위업무명]
- 업무 제목 3: 예정 내용

[단위업무명]
- 업무 제목 4: 예정 내용
```

## 7. 임시저장, 저장, 수정 요구사항

일반사용자와 관리자는 작성 중인 내용을 저장할 수 있어야 한다.

### 임시저장

- 작성 중인 업무 항목을 임시저장할 수 있어야 한다.
- 임시저장 상태의 항목은 최종 제출 항목으로 간주하지 않는다.
- 사용자는 임시저장 항목을 다시 불러와 수정할 수 있어야 한다.

### 저장

- 사용자는 업무 항목을 저장할 수 있어야 한다.
- 저장된 항목은 작성자 본인이 조회할 수 있어야 한다.
- 저장된 항목은 제출 대상으로 선택할 수 있어야 한다.

### 수정

- 사용자는 본인이 작성한 업무 항목을 수정할 수 있어야 한다.
- 관리자는 팀원이 제출한 항목을 병합 화면에서 수정할 수 있어야 한다.
- 수정 시 수정일시가 갱신되어야 한다.

## 11. DB 필요성 판단

이 앱은 DB가 필요하다.

DB가 필요한 이유는 다음과 같다.

- 로그인과 회원가입이 필요하다.
- 사용자 role을 저장하고, 일반사용자/관리자 권한을 구분해야 한다.
- 일반사용자는 본인이 작성한 업무 항목을 저장, 수정, 조회해야 한다.
- 임시저장 기능은 브라우저를 닫아도 유지되어야 한다.
- 관리자는 여러 팀원이 제출한 항목을 기간별/팀원별/단위업무별로 조회해야 한다.
- 제출된 항목과 관리자가 수정한 병합 결과를 이력으로 남겨야 한다.
- CSV/Excel 업로드 결과를 사용자가 확인 후 저장하거나 수정할 수 있어야 한다.
- 금주/차주 구분, 제출 여부, 병합 결과 같은 상태값을 지속적으로 관리해야 한다.

DB 없이 구현 가능한 범위는 제한적이다.

- 단일 사용자가 브라우저에서 파일을 업로드하고 즉시 텍스트를 복사하는 일회성 도구라면 DB 없이도 가능하다.
- 하지만 로그인, role, 임시저장, 저장, 수정, 관리자 취합 기능이 포함되면 DB가 사실상 필수다.

권장 방향:

- MVP 단계부터 DB를 포함한다.
- 사용자, 업무 항목, 병합 보고서 테이블을 최소 모델로 시작한다.
- 파일 원본 자체를 반드시 DB에 저장할 필요는 없지만, 업로드로 생성된 업무 항목 데이터는 DB에 저장해야 한다.
- Excel 원본 파일 보관이 필요하면 파일 스토리지와 업로드 이력 테이블을 별도로 둔다.

## 13. 비기능 요구사항

- 사용자는 저장/수정/복사 성공 여부를 명확히 알 수 있어야 한다.
- CSV/Excel 업로드 오류는 사용자가 이해할 수 있는 메시지로 표시해야 한다.
- 병합 전 원본 항목과 병합 후 텍스트를 비교할 수 있어야 한다.
- 관리자는 많은 팀원의 항목을 빠르게 필터링할 수 있어야 한다.
- role 기반 접근 제어가 우회되지 않아야 한다.
- 비밀번호는 평문으로 저장하지 않아야 한다.
- 추후 Excel 업로드, PDF/Word 내보내기, 승인 워크플로우 확장을 고려할 수 있다.

## 14. 현재 복사된 코드 기준 참고사항

현재 복사된 코드는 `index.html` 단일 파일 기반의 정적 프로토타입이다.

- 로그인 기능은 아직 없다.
- 회원가입 및 role 선택 기능은 아직 없다.
- DB 저장 기능은 아직 없다.
- 임시저장/저장/수정 기능은 아직 없다.
- 관리자 화면은 아직 없다.
- CSV 업로드와 텍스트 병합의 초기 아이디어만 포함되어 있다.
- 한글 문자열 인코딩이 깨진 부분이 있어 구현 전 복구가 필요하다.
