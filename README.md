# Weekly Report App

주간업무보고 앱으로 전환 중인 Spring Boot + React/Vite 프로젝트입니다.

## 목표

- 팀원이 주간업무 항목을 직접 입력하거나 파일로 업로드한다.
- 팀원은 제출할 항목을 체크박스로 선택한다.
- 선택된 항목은 단위업무와 금주/차주 기준으로 병합된다.
- 병합 결과는 텍스트로 미리보고 복사할 수 있다.
- 팀장은 팀원이 제출한 항목을 조회하고 병합 결과를 수정/복사한다.

## 현재 구현 상태

- Spring Boot 백엔드 기본 구조
- 공통 `ApiResponse`와 전역 예외 처리
- 회원가입, 로그인, JWT 발급
- JWT 인증 필터와 `/api/me`
- 팀원 주간업무 항목 생성, 조회, 수정, 제출
- 팀장 취합용 제출 항목 조회 API
- 병합 결과 저장, 조회, 수정 API
- React/Vite 로그인/회원가입 화면
- 팀원 제출 화면
  - 기준 일자 선택과 보고 기간 계산
  - 금주/차주 업무 항목 입력
  - 임시저장, 저장, 수정, 제출
  - 체크박스 선택
  - 단위업무별 병합 미리보기
  - 병합 결과 저장/수정 저장
  - 텍스트 복사
- 팀장 취합 화면
  - 제출된 팀원 업무 항목 조회
  - 기간/팀원/단위업무/주차 구분 필터
  - 체크박스 선택
  - 단위업무별 병합 결과 생성
  - 병합 결과 직접 수정
  - 병합 결과 저장/수정 저장
  - 최종 텍스트 복사

## 관리자 승인 정책

관리자 즉시 가입은 허용하지 않습니다.

회원가입 시 `MANAGER`를 요청하면 계정은 다음 상태로 생성됩니다.

- `role = USER`
- `requestedRole = MANAGER`
- `roleApprovalStatus = PENDING`

MVP 단계에서는 관리자 승인을 별도 화면/API로 만들지 않고 DB에서 수동으로 처리합니다.

승인 시 운영자가 DB에서 다음 값으로 변경합니다.

- `role = MANAGER`
- `roleApprovalStatus = APPROVED`

팀장 취합 API는 실제 `role`이 `MANAGER`인 사용자만 접근할 수 있게 구현합니다.

## 실행 방법

### 백엔드

Windows PowerShell:

```powershell
.\mvnw.cmd spring-boot:run
```

또는 IntelliJ IDEA에서 `WeeklyReportApplication`을 실행합니다.

로컬 개발에서는 `JWT_SECRET`이 없어도 실행 시 임시 JWT 키를 생성합니다. 운영 환경에서는 환경변수 또는 외부 설정으로 32바이트 이상의 `JWT_SECRET`을 반드시 지정해야 합니다.

### 프론트엔드

```powershell
cd frontend
npm install
npm run dev
```

프론트엔드는 Vite 개발 서버(`http://localhost:5173`)에서 실행합니다. Vite 프록시가 `/api` 요청을 `http://localhost:8080` 백엔드로 전달합니다. 백엔드 API를 주소창에서 직접 열면 JWT 헤더가 없으므로 보호된 API는 401 응답을 반환합니다.

## 주요 API

### 인증

- `POST /api/auth/signup`
- `POST /api/auth/login`
- `GET /api/me`

### 팀원 업무 항목

- `POST /api/report-items`
- `GET /api/report-items`
- `PUT /api/report-items/{itemId}`
- `POST /api/report-items/submit`

### 팀장 취합

- `GET /api/admin/report-items`

### 병합 결과

- `POST /api/merged-reports`
- `GET /api/merged-reports?reportStartDate=YYYY-MM-DD&reportEndDate=YYYY-MM-DD`
- `PUT /api/merged-reports/{reportId}`

`mergeType` 값으로 일반사용자 병합 결과는 `MEMBER`, 팀장 취합 결과는 `ADMIN`을 사용합니다. `ADMIN` 병합 결과 저장은 실제 `MANAGER` 권한 사용자만 가능합니다.

## 병합 결과 저장 흐름

### 팀원

1. 로그인 후 기준 일자를 선택해 보고 기간을 확인한다.
2. 금주/차주 업무 항목을 입력한 뒤 `임시저장` 또는 `저장`한다.
3. 제출할 저장 항목을 체크박스로 선택한다.
4. 선택 항목의 단위업무별 병합 미리보기를 확인한다.
5. 병합 결과를 저장하거나 같은 화면에서 수정 저장한다.
6. 최종 텍스트를 복사해 주간업무보고에 사용한다.

### 팀장

1. `MANAGER` 권한으로 로그인한다.
2. 보고 기간, 팀원 아이디, 단위업무, 주차 구분으로 제출 항목을 조회한다.
3. 취합할 항목을 체크박스로 선택한다.
4. `병합`을 눌러 단위업무별 최종 텍스트를 생성한다.
5. textarea에서 병합 결과를 직접 다듬는다.
6. 병합 결과를 저장하거나 같은 화면에서 수정 저장한다.
7. 최종 텍스트를 클립보드에 복사한다.

저장된 병합 결과는 같은 보고 기간의 목록에서 다시 불러와 수정 저장할 수 있다. 병합 결과는 원본 업무 항목 ID를 함께 저장해 어떤 항목으로 생성되었는지 추적한다.

## 수동 테스트 체크리스트

- 팀원 계정으로 로그인한다.
- 업무 항목을 임시저장하고 목록에 남는지 확인한다.
- 저장한 항목을 수정한 뒤 다시 저장한다.
- 저장 상태의 항목만 선택 및 제출 가능한지 확인한다.
- 선택한 항목의 병합 미리보기가 금주/차주와 단위업무 기준으로 나뉘는지 확인한다.
- 병합 결과를 저장하고, 같은 선택 상태에서 다시 수정 저장할 수 있는지 확인한다.
- 팀장 계정으로 로그인한다.
- 제출된 팀원 항목을 기간/팀원/단위업무/주차 구분으로 조회한다.
- 선택 항목을 병합하고, 최종 텍스트를 수정한 뒤 저장한다.
- 저장된 취합 결과를 같은 화면에서 다시 수정 저장하고 복사할 수 있는지 확인한다.
